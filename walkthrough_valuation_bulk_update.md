# 評価額一括更新画面の新機能 Walkthrough

このドキュメントは、**評価額一括更新**画面に追加された以下の機能の確認手順を示します。

## 追加機能
1. **未入力の評価額入力欄を強調表示**
   - 入力が空の場合、黄色系の枠と背景色でハイライトされます。
2. **前回評価額をクリックで現在評価額にコピー**
   - 前回評価額セルをクリックすると、その金額が同行の入力欄に自動的に設定されます。
   - カーソルが変わり、ホバー時に文字色が変化し、クリック可能であることが視覚的に分かります。

## 確認手順
1. 開発サーバーを起動（`npm run dev`）し、ブラウザで `http://localhost:3000/assets/valuation` にアクセス。
2. **表示設定**で評価対象の資産が表示されていることを確認。
3. 各行の **現在評価額** 入力欄が空の場合、黄色の枠と薄いオレンジ背景が表示されていることを確認。
4. **前回** の金額セルにマウスを合わせると、文字色が `primary` に変わり、下線が表示されることを確認。
5. 前回金額セルをクリックすると、同行の入力欄にその金額が自動で入力され、ハイライトが解除されることを確認。
6. 必要に応じて他の行でも同様に動作することをテスト。
7. **一括保存** ボタンをクリックし、保存が成功することを確認（トーストで成功メッセージが表示される）。

## 注意点
- 入力欄が空のまま保存しようとすると、`isSaving` が無効になるため保存は行われません。
- 前回金額をコピーした場合でも、手動で上書き可能です。
- UI のハイライトは `valuations` オブジェクトにキーが無い（未入力）場合に適用されます。

以上の手順で新機能が正しく動作することを確認してください。
